This app will provide users access to information about movies, directors, and genres. Users will be able to sign up, update their profile, and create a list of favorite movies.

In creating this app, I have used Node.js and the PowerShell to create the folder, as well as each file. I've used Express to run the server, UUID to auto create user ID numbers.

    1. Return a list of all movies to user.
  • description: returns complete array of movies
  • endpoint url: localhost:5000/movies
  • method type: GET
  • paramenters in url: movies
  • format of data expected in body of request: list of movies
  • format of response data: json
    2. Return data about a single movie by title to user.
  • description: returns one movie's complete info
  • endpoint url: localhost:5000/movies/Harry%20Potter%20and%20the%20Prisoner%20of%20Azkaban
  • method type: GET
  • paramenters in url: movies/:Title
  • format of data expected in body of request: one movie and all info of that movie
  • format of response data: json
    3. Return data about a genre by name.
  • description: returns array of movies with specific genre
  • endpoint url: localhost:5000/movies/genre/Fantasy
  • method type: GET
  • paramenters in url: movies/:Genre/:Name
  • format of data expected in body of request: returns list of movies with specified genre
  • format of response data: json
    4. Return data about a director by name.
  • description: returns one director's complete info
  • endpoint url: localhost:5000/movies/director/David%20Yates
  • method type: GET
  • paramenters in url: movies/:Director/:Name
  • format of data expected in body of request: returns one director and all info about him
  • format of response data: json
    5a. Returns full list of users.
    5. Allow new users to register.
  • description: checks if user already exists, if not will new user registers, is auto assigned ID
  • endpoint url: localhost:8888/users
  • method type: POST
  • paramenters in url: /users
  • format of data expected in body of request: user name, bday, email, pw, id
  • format of response data: json
    6. Allow users to update their user info.
  • description: users can update their info
  • endpoint url: localhost:8888/users/1
  • method type: PUT
  • paramenters in url: users/:Username
  • format of data expected in body of request: user name, favoriteMovies, id
  • format of response data: json
    7. Allow users to add a movie to their list of favorites.
  • description: users add movies to favoriteMovies
  • endpoint url: localhost:8888/users/1/Harry%20Potter%20and%20the%20Prisoner%20of%20Azkaban
  • method type: POST
  • paramenters in url: users/:Username/movies/:MovieID
  • format of data expected in body of request: confirms to user that movie is added to list
  • format of response data: html
    8. Allow users to remove a movie from their list of favorites.
  • description: user can remove favoriteMovies
  • endpoint url: localhost:8888/users/1/The%20Prisoner%20of%20Azkaban
  • method type: DELETE
  • paramenters in url: users/movies
  • format of data expected in body of request: confirms to user that movie has been removed or was not found
  • format of response data: html
    9. Allow existing users to deregister.
  • description: removes user
  • endpoint url: localhost:8888/users/1
  • method type: DELETE
  • paramenters in url: users/:Username
  • format of data expected in body of request: confirms to user that user has been removed or not found
  • format of response data: html
